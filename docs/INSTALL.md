# 安装与试跑说明

## 1. 安装插件

1. 打开 Chrome
2. 访问：`chrome://extensions/`
3. 右上角打开「开发者模式」
4. 点击「加载已解压的扩展程序」
5. 选择项目目录：

```text
/Users/haoqi/OnePersonCompany/tempProject/20260501-doEcommerceTag
```

## 2. 导入 Excel / CSV

点击浏览器右上角插件图标，导入包含以下表头的文件。空值/脏数据不会阻断导入，会在结果报告里标记异常并跳过执行：

```csv
商家编码,编码
A001店,D401
A002店,D402
```

示例文件：

```text
samples/live-room-mapping.csv
```

## 3. 打开抖店商品创建页

目标页面类似：

```text
https://fxg.jinritemai.com/ffa/g/create?... 
```

进入商品发布/编辑页后，插件会在右侧注入按钮：

```text
批量直播间规格
```

## 4. HTTP 请求录制流程

如果 DOM 自动化效果不稳定，优先用请求录制来分析真实保存接口：

1. 重新加载插件并刷新抖店页面
2. 打开插件面板
3. 点击「开始录制请求」
4. 人工创建 1 个颜色分类自定义值，例如 `F820`
5. 人工填对应商家编码，例如 `B064`
6. 手动点击保存草稿或触发页面保存请求
7. 点击「停止录制」
8. 点击「导出请求」得到 JSON
9. 用该 JSON 分析接口 URL、payload 字段、签名/风控参数

## 5. 建议试跑顺序

第一次不要直接跑全部。

建议顺序：

1. 点击「诊断页面」
2. 点击「扫描预览」
3. 执行数量选择「先试 1 条」
4. 点击「开始执行」
5. 人工确认：
   - 是否在“颜色分类”下创建了 Excel「编码」列里的直播间编号
   - 是否选中了新建的直播间编号
   - 下方 SKU 表是否出现对应行
   - 商家编码是否写入正确
6. 如果 1 条成功，再试 5 条
7. 5 条成功后，再试 10 条
8. 最后再考虑全部执行

## 6. 当前安全边界

插件不会：

- 点击「发布商品」
- 点击「保存草稿」
- 绕过验证码
- 请求抖店后端接口
- 写 Cookie
- 做无人值守发布

## 7. 如果执行失败

优先点击「诊断页面」，把面板里的诊断 JSON 发回来，用来精修选择器。
